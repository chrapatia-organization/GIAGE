34 // the solver is a RK order 4, but instead of weighting the intermediate derivative by the usual 1/6, 1/3, 1/3, 1/6 coefficients
35 // I am using 1/4, 1/4, 1/4, 1/4.
36 // This is correct.The weighting coefficients of any RK method comes from an arbitrary criteria
37 // solving for a set of linear equation on the coefficients.
38 // The standard coefficients just happen to lead to an accurate result because they are optimal
39 // with respect to a second order solution.For differential equations of higher order, then is not
40 // clear if the traditional weigh factors are any better than any other set of weighting factors.
41 // A different set of coefficient generates results that are not much different than the optimal set,
42 // but it allows for simpler calculation of the intermediate derivatives and also for less intermediate memory.
43 // For more detail on the derivation of the Runge Kutta coefficients you can go to:
44 // http://pathfinder.scar.utoronto.ca/~dyer/csca57/book_P/node51.html

169 inline ndBodyKinematic* ndDynamicsUpdate::FindRootAndSplit(ndBodyKinematic* const body)
170 {
171  ndBodyKinematic* node = body;
172  while (node->m_islandParent != node)
173  {
174  ndBodyKinematic* const prev = node;
175  node = node->m_islandParent;
176  prev->m_islandParent = node->m_islandParent;
177  }
178  return node;
179 }
